# Мой опыт в Джете
## HR-ассистент для автоматизации подбора кандидатов

**Роль:** Solution Architect / ML Engineer / Backend Developer

Спроектировал и разработал внутренний HR-ассистент для GPTLab, который принимает описание вакансии в свободной форме, формирует структурированный профиль кандидата, обогащает его через RAG по базе должностных инструкций и возвращает HR ранжированный shortlist кандидатов. Отвечал за архитектуру сервиса, разработку ключевых модулей, настройку AI-агентов, RAG-контура и интеграцию микросервиса в пользовательский интерфейс.

**Ключевые результаты и задачи:**

- Спроектировал архитектуру HR-микросервиса и реализовал end-to-end пайплайн: свободный текст HR -> валидация и формализация требований -> генерация JSON-профиля -> обогащение профиля через RAG -> поиск кандидатов -> ранжирование резюме по релевантности.
- Разработал набор AI-агентов для отдельных этапов пайплайна: `Validator`, `Rephraser`, `Verboser`, `JSONAgent`, `Concatenator`, `ProfileGenerator`; разделил роли агентов, что упростило поддержку сценариев, доработку промптов и контроль качества результатов.
- Настроил RAG-контур для работы с корпоративной базой должностных инструкций: ingestion документов, typed-text представление профилей, retrieval через RagFlow, восстановление `CandidateProfile` из найденных документов и обогащение пользовательского профиля релевантными требованиями.
- Реализовал интеграционный слой поиска кандидатов через HH API: построение поисковых параметров, выгрузка shortlist резюме, получение детальных карточек кандидатов, преобразование данных в единый профиль и возврат отсортированного списка кандидатов в интерфейс.
- Разработал гибридный ранкер кандидатов на основе структурного и векторного сравнения профилей: объединил field-level matching, embeddings и эвристические веса, благодаря чему система ранжирует кандидатов сверху вниз по близости к целевому профилю.
- Подготовил сервисный контур для работы в интерфейсе: FastAPI-интеграция, Celery worker, Docker-сборка, env-конфигурация, обработка ошибок, файловый вывод результатов и сценарии ручной/интеграционной проверки.
- Вёл проект как основной разработчик: декомпозировал систему на модули, проектировал контракты данных, определял формат взаимодействия между агентами, retrieval-слоем, коннекторами и ранкером, а также довёл решение до рабочего состояния для HR-команды.
- По экспертной оценке HR-команды, внедрение ассистента сократило время первичного поиска и формирования shortlist кандидатов примерно в 2,5 раза, уменьшило объём ручных рутинных операций на ~60% и повысило релевантность первичной выборки кандидатов на ~25%.

**Стек:** Python, FastAPI, Celery, Docker, LLM, RAG, RagFlow, AI Agents, Prompt Engineering, Embeddings, JSON Schema, RapidFuzz, HTTPX, Requests, RabbitMQ, Git, HH API.
